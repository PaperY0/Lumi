# Lumi 恋语新用户引导与问卷流程改造计划

更新时间：2026-07-11

## 总目标

建立稳定、可回显、可继续使用的用户流程：

```text
欢迎页 → 用户识别/登录 → 新用户资料建档 → 男生问卷 → 女生问卷
→ 当前阶段专项问卷 → 生成关系画像或开始使用 Lumi
```

老用户流程：

```text
欢迎页 → 本地识别为老用户 → 首页
```

## 九步执行计划

### 第 1 步：现状检查

目标：确认当前入口、守卫、数据表和问卷结果读取方式。

已完成结论：记录在 `docs/LUMI_PROJECT_CONTEXT.md`。本步骤不改业务代码。

验收：流程分支、数据来源、缺口和兼容约束都有明确记录。

### 第 2 步：新用户/老用户入口流程

目标：调整新用户完成顺序，老用户直接进入首页。

范围：`App.tsx`、欢迎页、资料页保存后的跳转、男女问卷完成后的跳转。

验收：新用户按资料 → 男生问卷 → 女生问卷 → 阶段专项问卷进入；老用户不被强制重复引导；刷新不丢流程。

### 第 3 步：资料页阶段必填和保存位置

目标：阶段只保留初识接触期、升温期、暧昧观察期；阶段必填；保存按钮靠近资料卡；保存后有明确成功反馈。

验收：未选阶段不能保存；资料保存成功提示出现；刷新资料和阶段仍然存在；阶段改变后题库同步变化。

### 第 4 步：问卷完成状态

目标：男女问卷和三份阶段问卷都显示完成勾选、完成文案和必要的完成时间。

验收：完成后立即更新；刷新后仍显示；阶段切换后各阶段状态独立；新用户显示“开始填写”，老用户显示“重新填写”。

### 第 5 步：阶段问卷连续导航

目标：完成页提供下一份问卷入口。

路径：

```text
我的相处方式 → 她的互动观察 → 关系节奏与边界
```

最后一份提供：返回专项问卷、生成关系画像、开始使用 Lumi。

验收：每个按钮跳到正确页面；答案已保存；返回和浏览器后退不丢失状态。

### 第 6 步：新用户完成条件

目标：只有资料、阶段、男女问卷和当前阶段三份专项问卷都完成后，才把用户视为完成首次引导。

验收：缺一项不能标记完成；完成后可以生成关系画像或进入首页；老用户重新进入直接首页。

### 第 7 步：关系画像生成入口

目标：画像读取双方资料、男女问卷、当前阶段三份专项问卷和聊天记录。

验收：画像显示当前阶段；三个阶段给出不同建议；资料不完整时可生成基础画像并明确提示缺少内容。

### 第 8 步：阶段专项问卷总览

目标：总览页显示当前阶段、三份问卷、完成状态、推荐顺序和总体完成度。

验收：按钮文案和勾选准确；已完成问卷可重新填写；阶段切换后状态和题库正确刷新。

### 第 9 步：全流程验收与提交

固定命令：

```bash
npm run test -- --run
npm run type-check
npm run build
```

手工验收：清空测试数据后完成新用户全流程；再以已有数据重新打开确认老用户直接进入首页；检查三个阶段分流、问卷回显、画像生成和首页入口。

验收通过后才执行：

```bash
git status
git add <明确文件>
git commit -m "feat: improve onboarding questionnaire flow"
git push origin main
```

## 总体完成顺序

先修流程状态，再修页面交互，最后补全画像和测试：

1. 固化现状和规则
2. 修正新老用户路由守卫
3. 修正资料阶段必填与保存反馈
4. 建立问卷完成状态
5. 增加问卷连续导航
6. 收紧新用户完成条件
7. 打通画像生成参考数据
8. 优化专项问卷总览
9. 全面测试、打开页面验收、提交推送
