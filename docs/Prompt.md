## **第三阶段：验收标准 Prompt**

修完后，让 Claude Code 用这份验收：

```text
请根据以下验收标准逐项检查当前项目。你需要打开相关文件阅读代码，并结合实现逻辑判断是否达标。如果某项不达标，请指出文件路径、原因和建议修改方式。

## P0 布局验收

1. 模拟对话页面外层容器必须铺满视口。
   - 标准：页面不整体上移，不留异常空白，主区域至少 min-height: 100vh。

2. 模拟对话页面聊天窗口必须稳定。
   - 标准：消息列表区域可滚动，输入框固定在聊天面板底部，不会被遮挡。

3. 模拟对话页面桌面端布局必须清晰。
   - 标准：左侧场景区、中间聊天区、右侧反馈区不重叠、不错位。

4. 模拟对话页面移动端必须可用。
   - 标准：布局改为上下排列，不横向溢出。

## P1 视觉验收

5. 所有主要卡片必须有 hover 上浮效果。
   - 标准：hover 时 translateY、scale、shadow、border highlight 至少有 3 项变化。

6. 恋爱宝典、设置、关系画像页面卡片必须使用统一卡片动效。
   - 标准：这些页面不能存在完全静态、无 hover 的主卡片。

7. 档案页面底部注意框必须和两张资料卡片整体宽度对齐。
   - 标准：注意框左边缘对齐左卡片，右边缘对齐右卡片，桌面端横跨两列，移动端单列铺满。

8. 颜色系统不能只有粉色。
   - 标准：全局颜色变量或样式中包含珍珠白、香槟金、雾紫、蜜桃光、暖象牙白、深梅子灰等辅助色，并在页面中实际使用。

9. 液态玻璃质感必须明显。
   - 标准：主要卡片具备半透明背景、backdrop blur、边缘高光、柔和阴影。

## P2 动效验收

10. 数字指标必须使用 CountUp。
    - 标准：互动热度、置信度、评分、消息数量等数字不能全部是静态文本。

11. 关键标题必须使用 BlurText。
    - 标准：首屏标题、页面主标题或关键标语至少有若干处使用 BlurText。

12. 卡片滚动进入动画必须存在。
    - 标准：卡片进入视口时有 opacity、translateY、scale 或 blur 的进入动画。

13. Emoji 必须更高级。
    - 标准：普通 emoji 不应大面积裸露使用，应替换为线性图标、玻璃徽章、渐变符号、SVG 或统一 icon token。

## 代码质量验收

14. 不应出现大量重复 card 样式。
    - 标准：尽量抽象为公共组件或公共 class。

15. 不应破坏原有页面功能。
    - 标准：首页、档案、关系画像、聊天导入、AI 分析、帮我回复、模拟对话、设置都能正常渲染。

16. 不应出现 TypeScript 或 lint 明显错误。
    - 标准：运行 npm run lint / npm run build 时无阻塞性错误。

请输出验收结果表：
- 验收项
- 是否通过
- 证据文件
- 问题说明
- 修复建议
```

# **给 Claude Code 的 Emoji 改造 Prompt**

```text
请检查项目中所有裸露使用的 emoji，并将其改造成更高级的图标系统。

要求：
1. 搜索所有 JSX/TSX/HTML 中的 emoji。
2. 输出使用位置和语义。
3. 新增或复用 IconBadge 组件。
4. 用 IconBadge 替换裸露 emoji。
5. IconBadge 使用 Liquid Glass 风格：
   - 半透明背景
   - 玫瑰金或雾紫边框
   - 柔和内发光
   - 极简线性图标
6. 不要大面积使用彩色系统 emoji。
7. 如果短期无法绘制复杂 SVG，可以先用高级符号替代：
   - ✦ insight
   - ◇ stage
   - ◌ notice
   - ∞ long-term
8. 保持整体 Apple-like、高级、克制。
```
