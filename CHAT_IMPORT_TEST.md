# 聊天导入功能测试指南

## 功能概述
实现了微信聊天记录导入功能，支持解析微信"复制"功能导出的真实三行一组格式。

## 已完成的文件

### 1. `src/lib/chatParser.ts` - 聊天记录解析器
- 解析微信三行一组格式（昵称/时间/内容）
- 支持多行内容自动合并
- 自动检测所有发送者昵称
- 支持用户指定"我"的昵称并映射 sender 字段

### 2. `src/types/chat.ts` - 类型定义更新
- 为 `ChatMessage` 添加了 `senderName?: string` 字段，用于保存原始昵称

### 3. `src/lib/db/repositories/chatRepo.ts` - 数据库操作更新
- 新增 `createSessionWithMessages` 方法，一次性创建会话并导入消息

### 4. `src/app/components/ChatImportPage.tsx` - 导入页面 UI
- **步骤 1**: 输入区域 + 可折叠的导入帮助面板
- **步骤 2**: 解析结果展示 + 选择"我"是谁（单选按钮）
- **步骤 3**: 预览前 20 条气泡 + 确认导入

## 关键特性

### 1. 导入帮助面板（默认展开）
包含以下引导内容：
- ⚠️ 微信限制说明（黄色警告框）
- 单条复制方法
- 多选复制方法（推荐）
- 中转粘贴技巧（解决微信→QQ 粘贴丢失问题）
- 超过 100 条的处理方法

### 2. 多行内容合并
正确处理跨多行的消息内容，如：
```
Whiskey
2026年06月23日 08:01
嗯呐
挺好的
嘿嘿嘿
```
会合并为一条消息，内容为："嗯呐\n挺好的\n嘿嘿嘿"

### 3. 昵称空格处理
自动 trim 每行，处理昵称前的空格（如 " Whiskey"）

### 4. 错误处理
- 格式异常自动跳过并提示
- 空内容自动过滤
- 昵称数量异常时给出提示

## 注意事项

1. **数据库版本未变更**：`senderName` 是可选字段，不需要修改数据库 schema 版本
2. **向后兼容**：旧数据没有 `senderName` 字段也能正常工作
3. **本地存储**：所有数据存储在浏览器 IndexedDB，不上传服务器
4. **占位符保留**：`[动画表情]` `[图片]` `[语音]` 等原样存储
